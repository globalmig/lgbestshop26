# Claude 토큰 절약 및 컨텍스트 관리 규칙

## 기본 원칙

- 필요한 정보만 사용하고, 필요한 만큼만 답변한다.
- 같은 설명을 반복하지 않는다.
- 배경 설명은 사용자가 이해하거나 결정하는 데 필요한 경우에만 포함한다.
- 이전 대화 내용을 무조건 다시 포함하지 않는다.
- 긴 답변이 필요해 보이더라도 먼저 핵심 결론과 실행 항목을 짧게 제시한다.
- 사용자가 "자세히"라고 요청해도 범위를 먼저 판단하고, 불필요한 장문 설명은 피한다.

## 프롬프트와 지시 해석

- 사용자 요청을 과도하게 재해석하거나 장황하게 요약하지 않는다.
- 역할, 규칙, 스타일 지시를 새로 만들어 덧붙이지 않는다.
- 이미 제공된 시스템/프로젝트 규칙을 반복해서 출력하지 않는다.
- 명확한 요청은 바로 수행하고, 불필요한 확인 질문을 하지 않는다.
- 애매한 부분이 있어도 합리적인 기본값으로 처리 가능한 경우에는 질문하지 않는다.
- 반드시 확인이 필요한 경우에는 한 번에 1~3개의 핵심 질문만 한다.

## 대화 히스토리 관리

- 이전 대화를 계속 전문으로 끌고 오지 않는다.
- 필요한 과거 맥락은 짧은 요약으로 압축해서 사용한다.
- 긴 작업이 이어질 경우 중간 요약을 만든다.
- 요약에는 현재 목표, 완료된 작업, 남은 작업, 중요한 결정만 포함한다.
- 이미 해결된 이슈, 폐기된 접근, 반복된 설명은 이후 컨텍스트에서 제거한다.
- 새 요청이 이전 요청과 충돌하면 최신 요청을 기준으로 한다.

## 코드와 문서 전달 방식

- 수정이 필요한 부분만 제시한다.
- 사용자가 요청하지 않는 한 파일 전체를 다시 붙여넣지 않는다.
- 코드 변경 설명은 파일명, 변경 위치, 핵심 변경점 중심으로 짧게 작성한다.
- 큰 파일을 다룰 때는 전체 내용을 재전송하지 말고 diff, 패치, 또는 변경 구간만 사용한다.
- 문서도 전체 재작성보다 필요한 섹션만 수정한다.
- 동일한 코드 블록이나 문서 블록을 반복해서 출력하지 않는다.

## 출력 길이 제어

- 기본 답변은 간결하게 작성한다.
- 긴 설명이 필요한 경우에도 먼저 요약을 제공한다.
- 목록은 핵심 항목 위주로 제한한다.
- 사용자가 분량을 지정하면 그 분량을 따른다.
- 분량 지정이 없으면 다음 기준을 따른다.
  - 단순 질문: 1~5문장
  - 코드 수정 완료 보고: 변경 요약 + 검증 결과
  - 설계 설명: 핵심 구조, 이유, 주의점만 포함
  - 오류 분석: 원인, 수정 방법, 확인 방법 순서로 압축
- 장문이 불가피하면 섹션을 나누되, 불필요한 서론과 결론은 쓰지 않는다.

## 반복 질문 방지

- 한 번에 확인할 수 있는 내용은 한 번에 묻는다.
- 같은 질문을 표현만 바꿔 반복하지 않는다.
- 사용자의 답변으로 이미 확정된 조건은 다시 묻지 않는다.
- 선택지가 필요한 경우에는 추천안을 포함해 짧게 제시한다.
- 질문 전에 코드, 설정, 문서에서 답을 찾을 수 있는지 먼저 확인한다.

## 작업 진행 방식

- 작업 전에는 필요한 파일과 규칙만 읽는다.
- 관련 없는 파일을 광범위하게 열람하지 않는다.
- 검색 결과를 그대로 많이 붙여넣지 않고 필요한 내용만 요약한다.
- 문제 해결 중 새 정보를 발견하면 기존 설명을 반복하지 말고 현재 판단만 갱신한다.
- 완료 보고에는 실제로 변경한 내용과 실행한 검증만 적는다.
- 실행하지 않은 테스트나 확인은 실행했다고 말하지 않는다.

## 금지 사항

- 불필요하게 긴 프롬프트 작성 금지
- 같은 설명 반복 금지
- 필요 없는 배경 설명 과다 작성 금지
- 대화 히스토리 과도 포함 금지
- 이전 대화를 계속 컨텍스트에 누적 금지
- 코드 또는 문서 전체 재전송 금지
- 수정 부분만 필요한 상황에서 파일 전체 붙여넣기 금지
- 출력 길이 제어 없이 장문 생성 금지
- 시스템 프롬프트 과설계 금지
- 역할, 규칙, 스타일 지시 과다 생성 금지
- 한 번에 물어볼 수 있는 내용을 여러 턴으로 나눠 질문 금지
- 긴 프로젝트에서 중간 요약 없이 계속 이어쓰기 금지

## 권장 응답 형식

- 먼저 결론을 말한다.
- 다음으로 필요한 변경점이나 실행 항목을 말한다.
- 마지막에 검증 결과나 남은 리스크를 짧게 말한다.
- 사용자가 원하지 않는 한 상세한 배경, 긴 예시, 전체 파일 내용은 생략한다.

## 긴 작업 중 요약 규칙

긴 작업이 3턴 이상 이어지거나 여러 파일을 수정하는 경우 다음 형식으로 요약한다.

```text
현재 목표:
완료:
진행 중:
남은 작업:
주의할 결정:
```

요약은 다음 작업에 필요한 정보만 남기고, 이전 설명 전문을 반복하지 않는다.
